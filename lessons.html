Lesson 3

Discussions
  • Cascading Style Sheet (CSS) designs webpage.
  • CSS can be written
    1. in line (writing CSS code in html tags; has the final say if there is any conflict of interest);
      uses the "style" attribute
    2. Internal CSS: written in the "style" tag (inside the "head" tag), just like html is written in the "body" tag.
      codes look like: selector {property:value;};
      uses the "script" element
    3. External CSS: uses the "link" element to reference the stylesheet (make sure you save the stylesheet as a CSS document!);
      might make code slower, but makes it easily reusable
  • rgb or hex values make you put in a value to get a specific color, go to "htmlcolorcode.com" to get the values you want. Can also get colors from Adobe lightroom, photoshop, etc
  • A style tag is an HTML element where the CSS code is written.
  • A document can have multiple style tags
  • "id" and "class" are HTML attributes that help us use CSS selectors
    "id" is unique and referenced by an #
    "class" is not unique and is referenced by a . (dot)

End of Lesson 3

Lesson 7

Discussions
  • Serif letters have projected tips, sans-serif means without serif
  • font-family is written in order of preference (usually 3)
  • google fonts is an amazing resource to pick nice fonts
  • Positioning:
    • Static: default positioning; not affected by 'top', 'bottom', 'right' or 'left' positioning
    • Relative: allows it (the element it's sed on) to be moved relative to the page based on the 'top', 'bottom', 'right' or 'left' positioning
    • Fixed: this allows an element to be fixed in a page, i.e it doesn't move even when you scroll up or down (like a chatbox)
    • Absolute: positioned relarive to it's parent
    • A positioned element is an element that not positioned the default (static) way
  • Positioning overwrites margins if comes after the margin statement.

End of Lesson 7
